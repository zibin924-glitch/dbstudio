# DBStudio — 数据库开发工具 需求规格说明书

## 项目概述

DBStudio 是一个基于 Python 的 Web 数据库开发工具，面向开发者和 DBA，提供数据库结构浏览、文档与代码生成、SQL 查询执行与展示、以及将 SQL 一键发布为 Web API 的能力。

**目标数据库**：MySQL、PostgreSQL、Oracle 11g  
**技术栈**：Python（FastAPI + SQLAlchemy） + 前端框架（Vue 3 / React）  
**部署方式**：Docker 容器化，支持单机部署

---

## 一、连接管理模块

### 1.1 连接配置

用户通过 Web 界面填写数据库连接信息，系统根据配置建立连接。

**配置字段：**

| 字段 | 说明 | 示例 |
|------|------|------|
| 连接名称 | 用户自定义别名，用于快速识别 | 生产库-主库 |
| 数据库类型 | MySQL / PostgreSQL / Oracle 11 | MySQL |
| 主机地址 | IP 或域名 | 192.168.1.100 |
| 端口 | 数据库端口（提供默认值） | 3306 / 5432 / 1521 |
| 用户名 | 数据库登录账号 | root |
| 密码 | 数据库登录密码（加密存储） | •••••• |
| 数据库/Schema | 默认连接的库名 | my_project_db |
| 连接参数（可选） | 高级配置，如 SSL、字符集等 | charset=utf8mb4 |

### 1.2 连接生命周期

- **连接测试**：保存前先测试连通性，返回成功/失败及具体错误信息（如"认证失败"、"连接超时"、"数据库不存在"）。
- **连接保存**：密码使用 AES-256 加密后持久化到本地数据库（SQLite）。
- **连接池管理**：每个连接配置维护一个连接池（默认 5 个连接），支持手动调整池大小。
- **连接健康检查**：定时心跳检测，自动回收失效连接。
- **多连接并行**：支持同时保持多个不同数据库的连接，在界面上可快速切换。

### 1.3 连接分组与标签

- 支持对连接进行分组（如：开发环境、测试环境、生产环境）。
- 支持标签系统，便于检索和管理。

## 二、数据库结构浏览模块

### 2.1 数据库概览

连接成功后，左侧导航树展示数据库结构层级：

```
📁 连接名称
  ├── 📁 Schemas / Databases
  │   ├── 📁 schema_a
  │   │   ├── 📋 表 (Tables)
  │   │   ├── 👁️ 视图 (Views)
  │   │   ├── ⚡ 存储过程 (Procedures)
  │   │   ├── 🔧 函数 (Functions)
  │   │   ├── 🔄 触发器 (Triggers)
  │   │   └── 📦 序列 (Sequences) — PostgreSQL/Oracle
  │   └── 📁 schema_b
  └── 📊 数据库统计
```

**顶部统计面板**展示：表数量、总行数（估算）、总数据大小、索引大小。

### 2.2 表结构详情

点击某张表后，右侧展示完整的表信息，分为多个 Tab：

**Tab 1 — 字段信息（Columns）**

| 展示项 | 说明 |
|--------|------|
| 字段名 | 列名 |
| 数据类型 | 完整类型（如 VARCHAR(255)、DECIMAL(10,2)） |
| 可空 | YES / NO |
| 默认值 | 字段默认值 |
| 主键 | 是否为主键字段 |
| 自增 | 是否自增（MySQL AUTO_INCREMENT / PostgreSQL SERIAL） |
| 注释 | 字段备注（COMMENT） |
| 字符集 | 字符集与排序规则（MySQL） |

支持字段排序（按名称、类型）、搜索过滤。

**Tab 2 — 索引信息（Indexes）**

展示所有索引：索引名、类型（PRIMARY / UNIQUE / INDEX / FULLTEXT）、包含字段、排序方向、基数（Cardinality）。

**Tab 3 — 外键关系（Foreign Keys）**

展示外键约束：约束名、源字段、目标表、目标字段、更新规则（ON UPDATE）、删除规则（ON DELETE）。

**Tab 4 — 表属性（Properties）**

展示：表引擎（InnoDB/MyISAM）、字符集、排序规则、行数（估算）、数据大小、索引大小、创建时间、最后修改时间、表注释。

**Tab 5 — 数据预览（Preview）**

默认 `SELECT * FROM table LIMIT 100`，展示前 100 行数据，支持排序和简单过滤。

### 2.3 视图 / 存储过程 / 函数 / 触发器

点击后展示：名称、定义 SQL（格式化高亮）、创建时间、参数列表（存储过程/函数）。

### 2.4 ER 关系图（增强功能）

- 根据外键关系自动生成表关联图。
- 支持选择部分表生成局部 ER 图。
- 图表可拖拽、缩放。
- 支持导出为 PNG / SVG。

## 三、文档与代码生成模块

### 3.1 数据库文档生成

**输入**：选择要包含的表（全选或部分选）。  
**输出**：一份结构化的数据库设计文档。

**文档内容模板：**

1. **文档标题与版本信息**：数据库名、生成时间、版本号。
2. **目录**：自动生成的章节目录。
3. **数据库概要**：表数量、数据库类型、字符集等全局信息。
4. **表目录**：按 Schema 分组的表清单，每张表附简要说明。
5. **表详细设计**：每张表一个章节，包含字段列表、索引、外键、建表 DDL。
6. **ER 关系图**：嵌入生成的 ER 图。

**导出格式**：
- Markdown（`.md`）
- Word 文档（`.docx`）— 使用 python-docx
- PDF（`.pdf`）— 使用 WeasyPrint 或 ReportLab

### 3.2 DDL 生成

- **导出建表语句**：生成 `CREATE TABLE` 语句，支持选择是否包含索引和外键。
- **导出整个 Schema**：所有表的 DDL 合并为一个 `.sql` 文件。
- **数据库间转换（增强功能）**：将 MySQL DDL 转换为 PostgreSQL / Oracle 兼容语法（类型映射、语法差异处理）。

### 3.3 代码生成

根据表结构自动生成代码模板，支持以下目标：

| 生成目标 | 说明 |
|----------|------|
| SQLAlchemy Model | Python ORM 模型，含字段类型映射、关系定义 |
| Django Model | Django ORM 模型 |
| Pydantic Schema | 用于 FastAPI 的请求/响应模型 |
| TypeScript Interface | 前端类型定义 |
| Go Struct | Go 语言结构体，含 GORM tag |
| Java Entity | JPA / MyBatis 实体类 |

**代码生成配置：**
- 选择目标语言/框架。
- 命名风格转换（snake_case → camelCase / PascalCase）。
- 可选包含注释映射。
- 支持自定义 Jinja2 模板扩展。

## 四、SQL 查询控制台模块

### 4.1 SQL 编辑器

- **语法高亮**：使用 Monaco Editor（VSCode 同款编辑器），支持 SQL 语法高亮。
- **自动补全**：基于当前连接的数据库结构，提供表名、字段名、关键字的智能补全。
- **多 Tab 支持**：可同时打开多个查询标签页，各自独立执行。
- **SQL 格式化**：一键格式化 SQL（使用 sqlparse / sqlfluff）。
- **快捷键**：`Ctrl+Enter` 执行，`Ctrl+Shift+F` 格式化。

### 4.2 查询执行与结果展示

- **执行引擎**：通过连接池发送 SQL 到目标数据库。
- **结果表格**：以可排序、可筛选的表格展示查询结果。
- **分页**：大数据集自动分页加载（默认每页 50 行）。
- **数据类型渲染**：日期、JSON、大文本等类型做特殊渲染（如 JSON 可折叠展开）。
- **执行统计**：显示执行耗时、影响行数、扫描行数。
- **多结果集**：存储过程返回多个结果集时，分别展示。
- **只读保护**：可配置"只读模式"，阻止 `INSERT / UPDATE / DELETE / DROP` 等写操作。

### 4.3 查询历史

- 自动保存执行过的 SQL 及执行时间、耗时、结果行数。
- 支持按关键词搜索历史记录。
- 支持收藏常用 SQL，方便复用。

### 4.4 结果导出

- 导出为 CSV / Excel（`.xlsx`）/ JSON。
- 可选择导出全部数据或当前选中行。
- 大数据量导出采用流式写入，避免内存溢出。

### 4.5 执行计划（增强功能）

- 对 `SELECT` 语句生成 `EXPLAIN` 执行计划。
- 以表格或树形结构展示执行计划各节点。
- 高亮全表扫描、文件排序等性能问题。

## 五、SQL 发布为 Web API 模块

这是本工具的特色功能——用户编写好一条 SQL 查询后，可以将其"发布"为一个可被外部调用的 HTTP 接口。

### 5.1 API 定义

用户通过界面配置以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API 名称 | 接口名称 | get-user-orders |
| HTTP 方法 | GET / POST | GET |
| URL 路径 | 对外暴露的路由 | /api/v1/user-orders |
| SQL 模板 | 带参数占位符的 SQL | `SELECT * FROM orders WHERE user_id = :user_id AND status = :status` |
| 参数定义 | 每个参数的名称、类型、是否必填、默认值 | user_id: int, 必填; status: string, 默认='paid' |
| 所属连接 | 该 SQL 执行的目标数据库 | 生产库-主库 |
| 结果限制 | 最大返回行数 | 1000 |
| 缓存策略 | 结果缓存时间（可选） | 60 秒 |

### 5.2 参数绑定与安全

- **参数化查询**：所有用户传入的参数通过绑定变量方式传入，杜绝 SQL 注入。
- **类型校验**：按定义的参数类型进行校验，类型不匹配时返回 400 错误。
- **白名单机制**：只允许发布时定义的 SQL 被执行，不允许任意 SQL。

### 5.3 API 认证与限流

- **Token 认证**：每个 API 可生成独立的访问 Token，也可使用全局 API Key。
- **IP 白名单**（可选）：限制允许访问的 IP 范围。
- **限流**：可配置每分钟/每秒最大请求数。

### 5.4 API 运行与监控

- **在线测试**：在界面上填写参数、发送请求、查看返回结果。
- **自动生成文档**：每个 API 自动生成 OpenAPI/Swagger 文档，可在线调试。
- **调用统计**：记录每个 API 的调用次数、平均响应时间、错误率。
- **日志**：记录每次 API 调用的请求参数、响应时间、状态码。

### 5.5 API 生命周期管理

- **启用 / 禁用**：一键开关，禁用后返回 503。
- **版本管理**：支持同一 API 的多个版本共存（如 /v1/... 和 /v2/...）。
- **变更历史**：记录 SQL 模板和配置的每次修改。

### 6.1 技术选型

| 层级 | 技术 | 说明 |

## 七、非功能性需求

### 7.1 安全性

- **密码加密**：数据库连接密码使用 AES-256 加密存储，密钥由环境变量注入。
- **SQL 注入防护**：API 模块中所有用户参数通过参数化查询绑定，不允许拼接 SQL。
- **权限隔离**：支持配置只读账号，防止误操作。
- **审计日志**：记录关键操作（连接创建/删除、API 发布/修改、DDL 导出）。

### 7.2 性能

- **连接池复用**：避免频繁建立/断开数据库连接。
- **查询超时**：可配置 SQL 执行超时时间（默认 30 秒），超时自动取消。
- **大结果集分页**：查询结果默认分页返回，避免一次性加载百万行。
- **异步执行**：文档生成、数据导出等耗时任务异步处理，不阻塞界面。

### 7.3 可用性

- **响应式布局**：适配不同屏幕尺寸。
- **暗色模式**：支持亮色/暗色主题切换。
- **国际化**：预留 i18n 支持（中文/英文）。

### 7.4 部署

- **Docker 一键部署**：`docker-compose up` 即可启动完整服务。
- **环境变量配置**：加密密钥、端口、日志级别等通过环境变量管理。
- **数据持久化**：连接配置、历史记录等持久化到挂载卷。

## 九、关键交互流程

### 9.1 首次使用流程

```
用户打开 Web 界面
  → 引导创建第一个数据库连接
  → 填写连接信息 → 测试连接 → 保存
  → 自动进入结构浏览页面
  → 左侧展示数据库树形结构
```

### 9.2 SQL 发布为 API 流程

```
用户在 SQL 控制台中编写并调试好一条查询
  → 点击"发布为 API"按钮
  → 系统预填 SQL 模板，用户补充：
      - API 名称和路径
      - 参数定义（系统自动从 SQL 中提取占位符）
      - 认证方式
  → 确认发布
  → 系统生成 API 端点和文档
  → 用户可在线测试或复制 URL 分享给他人
```

## 十、技术风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| Oracle 11 驱动兼容性 | cx_Oracle 安装依赖 Oracle Client | 使用 `oracledb`（纯 Python 模式），或在 Docker 中预装 Instant Client |
| 不同数据库元数据查询差异 | INFORMATION_SCHEMA 在不同数据库中不统一 | 每个方言模块独立实现元数据查询，SQLAlchemy Inspector 作为基础抽象 |
| SQL-to-API 安全性 | 暴露 Web 接口可能被滥用 | 参数化查询 + Token 认证 + 限流 + 只允许 SELECT |
| 大表结构加载慢 | 字段极多的表展示卡顿 | 懒加载 + 虚拟滚动 + 元数据缓存 |
| 前端 SQL 编辑器性能 | 大 SQL 文本编辑卡顿 | Monaco Editor 本身性能优秀，注意补全数据量控制 |
